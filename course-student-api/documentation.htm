POST /login/signup

-          Details: Create a new user account, with authentication credentials stored in Auth0 database and user details in Google Datastore.

-          Request:

o   Header: Content-Type (only accepts application/json)

o   Body input in JSON: email (a string), password (a string), full_name (a string), department (a string), and start_date (a string).

  Note: email must be a properly formatted email string, and the remaining properties must be non-empty strings.

  Example:

{

"email": "bob@example.com",

"password": "abc123",

"full_name": "Bob Fisher",

"department": "Math",

"start_date": "1/1/2010"

}

-          Response:

o   Valid input: Returns status code 201 Created, a link to the newly created resource in the Location header, and a JSON object in the body with the following properties: full_name, department, start_date, email, owner_of_courses, id, and self.

  Example in Location header:

Location: https://cs493-final-project-derrick.appspot.com/users/bob@example.com

  Example in body:

{

"full_name": "Bob Fisher",

"department": "Math",

"start_date": "1/1/2010",

"email": "bob@example.com",

"owner_of_courses": [],

"id": "5698529657880576",

"self": "https://cs493-final-project-derrick.appspot.com/users/bob@example.com"

}

o   Invalid body input: Returns status code 400 Bad Request, and error message in body.

  Example, any empty property:

The following properties are required: email, password, full_name, department, and start_date.

  Example, using an email that is already in use:

{

"name": "BadRequestError",

"code": "user_exists",

"description": "The user already exists.",

"statusCode": 400

}

  Example, an invalid email:

{

"error": "invalid email address"

}

o   Invalid Content-Type header: Returns status code 415 Unsupported Media Type, and error message in body.

  Example, Content-Type header set to application/xml:

Server can only accept 'application/json'.

o   Invalid Accept header (i.e. any other value besides application/json): Returns status code 406 Not Acceptable, and error message in body.

  Example, Accept header set to application/xml:

Server can only return 'application/json'.

 

POST /login

-          Details: Log into a user account to retrieve a JSON Web Token (JWT) using Auth0 authentication.

-          Request:

o   Header: Content-Type (only accepts application/json)

o   Body input in JSON: username (a properly formatted email address string), password (a non-empty string).

  Example:

{

"username": "bob@example.com",

"password": "abc123"

}

-          Response:

o   Valid input: Returns status code 200 OK and a JSON object in the body with the following properties: access_token, id_token, scope, expires_in, and token_type.

  Example in body:

{

"access_token": "CjE...A3J",

"id_token": "eyJ0...chg",

"scope": "openid profile email address phone",

"expires_in": 86400,

"token_type": "Bearer"

}

o   Invalid body input: Returns status code 400 Bad Request, and error message in body.

  Example, an incorrect username and/or password:

{

"error": "invalid_grant",

"error_description": "Wrong email or password."

}

  Example, an empty username:

{

"error": "invalid_request",

"error_description": "Missing required parameter: username"

}

  Example, an empty password:

{

"error": "invalid_request",

"error_description": "Missing required parameter: password"

}

o   Invalid Content-Type header: Returns status code 415 Unsupported Media Type, and error message in body.

  Example, Content-Type header set to application/xml:

Server can only accept 'application/json'.

o   Invalid Accept header (i.e. any other value besides application/json): Returns status code 406 Not Acceptable, and error message in body.

  Example, Accept header set to application/xml:

Server can only return 'application/json'.
